### 2026-04-03: Qiita API + fzf + rich + ポーリング

- **発見:** `capture_output=True` はstdoutとstderr両方を奪う。fzfはUIの描画にstderrを使うため、`capture_output=True` だとfzfのUIが表示されない。
- **判断:** `stdout=subprocess.PIPE` のみ指定し、stderrはターミナルに流す。
- **理由:** fzfはstderrでUIを描画し、stdoutで選択結果を返す設計のため。

- **発見:** Qiita APIは応答が速い（約0.6秒）。httpx + asyncioで問題なく動作する。
- **判断:** httpxのtimeoutは10秒で十分。
- **理由:** APIが遅延した場合の保険として設定。

- **発見:** richのMarkdownレンダリングは見出し・コードブロック・リストは整形されるが、画像・数式は非対応。
- **判断:** 画像を含む記事はブラウザ（wslview）で開く選択肢を提供。現状はQiitaのURLをそのまま開く。
- **理由:** ターミナルの技術的限界。ローカルHTML生成は将来の拡張とする。

- **発見:** ポーリングの差分検知はIDのsetをメモリ保持するだけで十分動作する。
- **判断:** PoCではメモリのみ。本実装では永続化（ファイルorDB）を検討。
- **理由:** 再起動すると既読情報が消えるため。
